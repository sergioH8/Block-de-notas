Acciones

En esta sección, describimos las acciones que puede realizar el usuario en la aplicación.

  • crear_accion_ayuda(): Crea un QAction que activa un modo de ayuda en la aplicación. Al hacer clic en cualquier opción, el usuario recibe una explicación sobre su funcionamiento, mejorando la comprensión del uso de la aplicación.
  • crear_accion_nuevo_bloc(): Crea un QAction que permite al usuario crear un nuevo bloc de notas. Este acción se conecta a un método que inicializa un espacio en blanco para que el usuario comience a trabajar.
  • crear_accion_abrir_archivo(): Crea un QAction que facilita al usuario abrir un archivo previamente guardado. Este acción conecta al método adecuado que permite acceder a documentos y datos almacenados en la aplicación.
  • crear_accion_guardar_archivo(): Crea un QAction que permite al usuario guardar los cambios realizados en el archivo actual. Se conecta a un método que asegura que el trabajo del usuario no se pierda.
  • crear_accion_deshacer(): Crea un QAction que ofrece la opción de revertir la última acción realizada. Se conecta a un método que permite al usuario corregir errores rápidamente.
  • crear_accion_rehacer(): Crea un QAction que permite al usuario volver a aplicar una acción que fue deshecha. Esta acción facilita la gestión de cambios realizados en el documento.
  • crear_accion_cortar(): Crea un QAction que permite al usuario eliminar una selección de texto o elementos, copiándolos al portapapeles para su uso posterior en otra ubicación.
  • crear_accion_copiar(): Crea un QAction que copia la selección de texto o elementos al portapapeles sin eliminarlos, permitiendo al usuario pegarlos en otra parte del documento.
  • crear_accion_pegar(): Crea un QAction que inserta el contenido almacenado en el portapapeles en la ubicación actual del cursor en el documento, facilitando la incorporación de texto.
  • crear_accion_buscar(): Crea un QAction que permite al usuario buscar texto dentro del documento, facilitando la localización de información relevante.
  • crear_accion_reemplazar(): Crea un QAction que permite al usuario reemplazar una palabra o frase específica en el documento con otra. Se conecta a un método que facilita la edición de texto.
  • crear_accion_reemplazar_todos(): Crea un QAction que reemplaza todas las instancias de una palabra específica en el documento. Esta acción agiliza el proceso de edición en textos largos.
  • crear_accion_color_de_bloc(): Crea un QAction que conecta a través de triggered.connect al método de cambiar color, añadiendo ayuda con .setWhatsThis y un QKeySequence usando .setShortcut para acceder más rápido a ese método.
  • crear_accion_salir(): Crea un QAction que cierra la aplicación, asegurando que se finalicen correctamente las sesiones de trabajo.

Botones

En esta sección, describimos las acciones que pueden realizar los botones de la aplicación.

  • crear_boton_nuevo_bloc(): Crea un QToolButton con un icono utilizando el método .setIcon. Este botón permitirá al usuario crear un nuevo bloc de notas, ofreciendo un espacio para organizar y almacenar información. Se añadirá a la barra de herramientas con el método .addWidget.
  • crear_abrir_archivo(): Crea un QToolButton con un icono utilizando .setIcon. Este botón facilita al usuario abrir un archivo previamente guardado, permitiendo el acceso a documentos almacenados en la aplicación. Se añadirá a la barra de herramientas usando .addWidget.
  • crear_boton_guardar_archivo(): Crea un QToolButton con un icono usando el método .setIcon. Este botón permite al usuario guardar los cambios realizados en el archivo actual, asegurando que su trabajo no se pierda. Se integrará a la barra de herramientas con .addWidget.
  • crear_boton_deshacer(): Crea un QToolButton con un icono utilizando .setIcon. Este botón ofrece la opción de revertir la última acción realizada, permitiendo al usuario corregir errores de forma rápida. Se añade a la barra de herramientas con .addWidget.
  • crear_boton_rehacer(): Crea un QToolButton con un icono utilizando .setIcon. Este botón permite volver a aplicar una acción que fue deshecha, facilitando la gestión de cambios en el documento. Se incluirá en la barra de herramientas usando .addWidget.

Métodos

En esta sección, describimos los métodos utilizados en la aplicación.

  • cambiar_color_de_bloc(): Este método permite al usuario cambiar el color de fondo del bloc de notas. Se puede acceder a este método desde el menú o desde un botón de herramientas. Al seleccionar un nuevo color, se actualizará el fondo de la ventana correspondiente.
  • abrir_archivo(ruta): Este método permite abrir un archivo en la aplicación. Recibe la ruta del archivo como argumento y carga su contenido en el editor de texto. Si el archivo no se puede abrir, se mostrará un mensaje de error al usuario.
  • guardar_archivo(ruta): Este método permite guardar el contenido del editor de texto en un archivo. Recibe la ruta como argumento y guarda los cambios realizados. Si la ruta es incorrecta o si ocurre un error durante el proceso, se mostrará un mensaje al usuario.

Widgets

En esta sección, describimos los widgets utilizados en la aplicación.

  • QTextEdit: Este widget permite la edición de texto enriquecido y es el principal componente utilizado para mostrar y modificar el contenido del bloc de notas.
  • QToolBar: Este widget se utiliza para crear la barra de herramientas de la aplicación. Contiene botones y acciones que facilitan el acceso a las funcionalidades de la aplicación.
  • QStatusBar: Este widget se utiliza para mostrar información sobre el estado actual de la aplicación. Puede mostrar mensajes temporales y el estado del archivo que se está editando.
  • QMenuBar: Este widget se utiliza para crear la barra de menú de la aplicación. Permite al usuario acceder a las acciones principales, como abrir, guardar y salir de la aplicación.
  • QWidget: Este widget es la base para todos los demás widgets en la aplicación. Se utiliza para crear la ventana principal y organizar otros widgets dentro de ella.
